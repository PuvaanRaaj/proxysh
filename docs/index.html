How it works

proxysh sets up in under a minute and runs silently in the background.

1

Generate a local CA

proxysh creates a Certificate Authority on your machine and installs it into your system keychain — one sudo prompt, done forever.

2

Add a domain

Run proxysh up myapp 3000. proxysh generates a TLS certificate for myapp.test, adds it to /etc/hosts, and reloads the proxy instantly.

3

Open in your browser

Visit https://myapp.test — trusted HTTPS, green padlock, no warnings. Your app on localhost is now a real domain.

4

Share publicly (optional)

Run proxysh share --port 3000 to get a public URL at https://swift-river-1234.proxysh.show that anyone can visit.

Commands

Everything you need, nothing you don't.

proxysh start Set up certificates and start the daemon
proxysh up <name> <port> Add https://<name>.test → localhost:<port>
proxysh down <name> Remove a .test domain
proxysh list List all active domains and their status
proxysh share --port 3000 Create a public URL for a local port
proxysh logs -f Follow the proxy daemon logs
proxysh stop Stop the daemon
proxysh doctor Run health checks on your setup
